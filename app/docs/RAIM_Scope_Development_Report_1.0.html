§ 1 / 8개요 및 참조 논문

본 프로젝트는 Watanabe 외(2020)의 "Low-cost multi-modal microscope using Raspberry Pi" [1]에서 제시된 LED 어레이 기반 다중모드 현미경의 하드웨어 구성을 그대로 채택하면서, 실험실 사용성을 위한 본격 GUI · 한국어 UI · 실시간 ROI 기반 DPC · 갤러리 뷰어를 추가한 확장 구현이다.

참조 논문 핵심 내용

[1] Watanabe W., Maruyama R., Arimoto H., Tamada Y. "Low-cost multi-modal microscope using Raspberry Pi." Optik — International Journal for Light and Electron Optics 212 (2020) 164713. DOI: 10.1016/j.ijleo.2020.164713

논문은 다음을 시연한다:

  • 광학계: Olympus LMPlanFL 10× NA 0.25 + 150mm tube lens + Pi Camera Module (3280×2464, 1.12 μm 픽셀)
  • 광원: Pimoroni UNICORNHAT 8×8 RGB LED, 5mm pitch, LED↔시료 거리 50 mm
  • 4가지 모드: BF, DF, DPC (좌-우 / 상-하), Polarization
  • DPC 공식:
IDPC,LR = (IL − IR) / (IL + IR)   ,   IDPC,TB = (IT − IB) / (IT + IB)
  • 방향 의존성: 좌-우 vs 상-하 illumination으로 구조 방향성 강조
  • 해상도: 1951 USAF target 측정 ~4.4 μm, CTF > 0.95
본 프로젝트의 차별 가치 Watanabe 등의 원 논문은 코드 단편 + 광학계 검증에 초점을 둔 short note이다. 본 RAIM Scope는 동일한 광학·LED 설정에 대해 실험실 일상 사용을 위한 풀 GUI를 PyQt5로 구현한 것이다. 구체적으로 (a) 한국어 UI, (b) 3가지 색상 테마, (c) 실시간 ROI 드래그, (d) DPC 5단계 자동 시퀀스, (e) 갤러리·시퀀스 플레이어, (f) 백그라운드 썸네일 로더, (g) V4L2 자동 최적화(anti-flicker, 수동 노출), (h) Sense HAT 2 호환을 추가하였다.

§ 5 / 8주요 기술 결정

① UV + system-site-packages

Pi에서 PyQt5 / OpenCV / sense-hat을 pip로 빌드하면 수십 분 소요 및 ARM wheel 부재 위험. apt 시스템 패키지 + UV venv 상속으로 uv venv가 ~수초 만에 완료, 524 KB만 차지.

② V4L2 백엔드 명시

cv2 디폴트는 GStreamer 우선 시도 → 실패 → V4L2 폴백 (~0.2 s 지연). cv2.CAP_V4L2 명시로 즉시 V4L2, 시작 시간 단축.

③ 백그라운드 썸네일 worker

QPixmap의 동기 디코딩이 GUI thread를 블락하는 것이 단일 최대 병목. QThread + QObject worker + 시그널 emit 패턴으로 해결. 갤러리 즉시 응답, 썸네일은 ms 후 도착.

④ 슬라이더 sliderReleased 패턴

Sense HAT I2C 통신은 ~수십 ms. 슬라이더 valueChanged 매번 set_pixels() 호출하면 드래그 중 I2C 폭주 → freeze. sliderReleased에서만 적용, 드래그 중엔 라벨만.

⑤ 표시-처리 해상도 분리

LiveView/Preview는 항상 cv2.resize로 1280px 다운샘플 후 표시. 원본 frame은 ROI 좌표 계산과 DPC 처리에만 사용. 모든 비례 변환은 유지.

⑥ 모드 키 영문 / 표시 한글

내부 상태 ("DPC_LEFT")는 영문 키 유지하여 외부 API/저장 메타데이터 일관성. 사용자 표시("DPC ◀ 좌")는 _PATTERN_DISPLAY 매핑으로 분리.

§ 6 / 8현재 한계

1. cv2 Python MJPG 디코딩 7 fps 한계
Pi 5 ARM64에서 OpenCV Python은 5MP MJPG 디코딩을 ~150 ms/frame으로 처리 (HW 디코더 미사용). 디폴트 1MP로 회피했지만 5MP 라이브뷰는 6 fps. GStreamer pipeline (libcamerasrc + jpegdec → appsink) 도입 시 30 fps 가능 추정.
2. LED PWM banding 잔존 가능성
20 ms 셔터로 ≥100 Hz PWM 광원은 평균화되지만, Sense HAT 2의 정확한 PWM 주파수가 더 낮으면 띠 잔존 가능. RAIM_EXPOSURE_X100US=500(50 ms) 이상 권장.
3. 5MP 모드의 라이브 뷰 응답성
DPC 캡처 자체는 ROI 크롭이라 5MP에서도 OK이나, 라이브 뷰는 6-7 fps로 부드럽지 않음. 캡처 직전에만 5MP로 전환하는 hybrid 모드 필요.
4. 편광 모드 미구현
논문 §3.4의 polarization microscopy는 별도 편광판 하드웨어가 필요해 현재 미구현. UI 자리만 마련하면 편광 메타데이터 기록은 가능.
5. 자동 초점·스테이지 미지원
수동 초점만 사용. 모터 stage + autofocus 알고리즘 필요.

S5 5MP hybrid 캡처 자체 한계 #3 라이브뷰는 1MP, "촬영 + 전체 저장" 시 일시적으로 5MP로 전환 후 복귀.
S6 QSettings 기반 설정 영속화 — 테마 선택, 출력 폴더, DF r 값, 노출/게인 마지막 값을 INI에 저장.
S7 Time-lapse 모드 Watanabe Ref [12] QTimer로 N분 간격 자동 DPC 촬영 → 시퀀스 폴더에 누적.

중기 (1–2 개월, 시스템 확장)

#제안근거구현 노트
M1 GStreamer libcamerasrc pipeline 자체 한계 #1 cv2 Python 우회 → 5MP @ 30 fps 라이브뷰 가능. Pi 5 HW JPEG 디코더 활용.
M2 WiFi 원격 모니터링 Watanabe §4 (Conclusion) FastAPI / WebSocket → 호스트 PC 브라우저로 라이브뷰 + 캡처. mDNS로 자동 발견.
M3 Fourier ptychography Watanabe Ref [15] (Tian 2014) 다수 LED 위치 캡처 + Fourier domain 합성 → 합성 고해상도. 처리 시간 분 단위, 별도 모듈로 분리.
M4 자동 초점 (contrast-based) — 모터 stage + Tenengrad/Brenner contrast metric 최대화 알고리즘.
M5 시야 stitching — 인접 시야 자동 모자이크 → 큰 시료 전체 영상. OpenCV stitcher API 활용.
M6 z-stack focus stacking — 다중 z-위치 캡처 후 라플라시안 가중 합성. 두꺼운 시료 전체 평면 인-포커스.

장기 (3 개월 이상, 응용 확장)

#제안구현 노트
L1 AI 분석 통합 세포 분할(StarDist/CellPose), 양파 분열기 자동 분류, 모기 stylet segmentation. ONNX 또는 PyTorch + Pi 5 NPU 가속.
L2 STEM 교육 모드 가이드 투어 (단계별 안내) + 시료 데이터베이스 + 정답 비교. Watanabe 논문의 핵심 동기 (STEM 교육).
L3 의료 진단 응용 혈액 도말, 기생충 검사 등. 규제 (CE/FDA) 고려 필요.
L4 클라우드 백업 및 협업 Google Drive / S3 자동 업로드 + 시퀀스 공유 링크 + 원격 분석.
L5 플러그인 시스템 커스텀 illumination 패턴, 후처리 필터, 분석 도구를 Python plugin으로 추가 가능하게.
