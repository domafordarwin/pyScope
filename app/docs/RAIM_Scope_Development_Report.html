§ 1 / 10개요 및 참조 논문

본 프로젝트는 Watanabe 외(2020)의 "Low-cost multi-modal microscope using Raspberry Pi" [1]에서 제시된 LED 어레이 기반 다중모드 현미경의 하드웨어 구성을 채택하면서, 실험실 사용성을 위한 본격 GUI · 한국어 UI · 실시간 ROI 기반 DPC · 갤러리 뷰어 · Hailo-10H NPU 기반 AI 검출 · 데이터셋 자동 생성 · Active learning까지 통합한 확장 구현이다.

참조 논문 핵심 내용

[1] Watanabe W., Maruyama R., Arimoto H., Tamada Y. "Low-cost multi-modal microscope using Raspberry Pi." Optik — International Journal for Light and Electron Optics 212 (2020) 164713. DOI: 10.1016/j.ijleo.2020.164713

논문은 다음을 시연한다:

  • 광학계: Olympus LMPlanFL 10× NA 0.25 + 150mm tube lens + Pi Camera Module
  • 광원: Pimoroni UNICORNHAT 8×8 RGB LED, 5mm pitch, LED↔시료 거리 50 mm
  • 4가지 모드: BF, DF, DPC (좌-우 / 상-하), Polarization
IDPC,LR = (IL − IR) / (IL + IR)   ,   IDPC,TB = (IT − IB) / (IT + IB)
V3 핵심 가치 — 학습 워크플로우 일체 통합
이전 V2는 Hailo NPU에서 사전 학습 모델(COCO 80 클래스)만 사용 가능했다. V3에서는 RAIM Scope 안에서 양파 분열기를 직접 라벨링하고 COCO/YOLO 데이터셋을 자동 생성하여, 외부 도구 없이 Ultralytics 학습 → Hailo 변환 → Pi 배포 → 실시간 검출의 전체 사이클을 닫았다. Active learning으로 AI 보조 라벨링도 가능해 데이터 수집 시간이 크게 단축된다.

P15

카메라 자동 분류 + V4L2 GUI 컨트롤 패널

FEATV3
  • 카메라 자동 분류 — USB ID 화이트리스트 + product name 키워드로 microscope(FIC OS-CM50, Cubeternet eSP570 등) vs general(일반 USB 웹캠) 자동 판정
  • 프로파일 자동 적용:
    • microscope: 수동 노출 20 ms + gain 24 + brightness 12 + exposure_dynamic_framerate=0 (Sense HAT LED 강광원 대응)
    • general: 자동 노출 + backlight_compensation=2 + dynamic framerate (실내 조명 / 역광 자동 적응)
  • CameraPanel 신규 위젯 — 카메라 시작 시 v4l2-ctl --list-ctrls 자동 파싱 → 사용 가능한 컨트롤만 슬라이더로 표시 (gain 없는 Realtek은 자동 숨김, 1-16 vs -64~64 등 범위 자동 적응)
  • 제어된 컨트롤: 자동 노출 토글 · 노출 시간 · 밝기 · 게인 · 대비 · 역광 보정 · 감마
  • cv2.set 우선 + v4l2-ctl fallback — 카메라 점유 중 v4l2-ctl이 silent fail하는 문제 해결 (cv2.set은 cap이 열린 상태에서 즉시 적용됨)
  • 슬라이더 sliderReleased 패턴 — I2C/USB 트래픽 폭주 방지

§ 5 / 10주요 기술 결정

① UV + system-site-packages

Pi에서 PyQt5/OpenCV/sense-hat/hailort/Python 모두 apt + UV venv 상속. 추가 pip 빌드 0초.

② cv2.set 우선 + v4l2-ctl fallback

cv2가 카메라 점유 중일 때 v4l2-ctl이 silent fail. cv2.set이 cap 열린 상태에서 즉시 적용. 슬라이더 → 영상 변경 즉시.

③ 카메라 자동 분류

USB ID + 이름 키워드로 microscope/general 판정 → 환경에 맞는 노출 프로파일 자동 적용. 사용자가 카메라 바꾸면 알아서 적응.

④ LiveView 공유 + 탭 분리

위쪽 LiveView는 항상 표시(영상 끊김 0), 아래쪽 탭으로 작업 모드 분리. 데이터셋 탭에서 라벨링하면서도 영상이 흐르는 것을 동시에 볼 수 있음.

⑤ Pending overlay 시각 구분

라벨링 박스 = 점선(📌). AI 박스 = 실선. 사용자가 "내가 찍은 박스"와 "AI가 찾은 박스" 를 즉시 구별.

⑥ Active learning seed

AI 검출 결과를 자동 라벨로 추가하지만, "스냅샷 + 라벨 저장" 호출 전까진 영구화 안 됨. 사용자가 검토할 시간 보장 — 잘못된 검출은 삭제/수정 가능.

⑦ train/val seed 고정

YOLO export 시 seed=42 → 같은 dataset에서 같은 분할. 모델 비교 실험에 재현성 확보.

⑧ COCO 표준 + GUI 결합

중간 형식을 표준 COCO JSON으로 → 다른 도구(Label Studio, FiftyOne 등)에서도 편집 가능. RAIM Scope에 종속되지 않음.

§ 6 / 10현재 한계

1. cv2 Python MJPG 디코딩 7 fps 한계
Pi 5 ARM64에서 5MP MJPG 디코딩이 ~150 ms/frame. 디폴트 1MP로 회피. GStreamer pipeline 도입 시 5MP @ 30 fps 가능.
2. AI 모델은 사전 학습이 COCO 80 클래스
내장 yolov11m_h10.hef는 person/car/cat 등만 검출. 양파 분열기는 §9 가이드 + §10 Colab 노트북으로 직접 학습 후 Hailo 변환 필요.
3. Active learning은 부트스트랩 모델 필요 V3
AI 보조 라벨링이 가치 발휘하려면 먼저 양파 분열기 모델이 있어야 함. 초기 30~50장은 수동 라벨링 → 모델 학습 → 그 후 AI 보조로 가속.
4. train/val 분할이 무작위 (계층화 X) V3
클래스별 비율이 작은 데이터셋(50장 미만)에서 한 클래스가 val에만 또는 train에만 몰릴 가능성. 향후 stratified split 추가 권장 (M9).

§ 9 / 10 ⭐ V3 갱신커스텀 모델 학습 가이드

V2에서는 외부 도구(Label Studio 등)가 필요했지만, V3는 RAIM Scope GUI 안에서 라벨링 + 데이터셋 export까지 한 번에 가능합니다. 학습만 외부 GPU(Colab Free 사용 가능)에서, 변환은 Hailo Dataflow Compiler에서.

5-step 단순화 워크플로우

  1. RAIM Scope에서 라벨링 (200-300장) — 라이브 탭에서 양파 BF로 분열기 발견 → 데이터셋 탭에서 클래스 선택 + ROI 박스 추가. Active learning 모드도 가능.
  2. YOLO 형식 export — 데이터셋 탭의 "📦 YOLO 형식으로 export" 버튼 → train/val 자동 80/20 분할 + data.yaml 자동 생성.
  3. Colab에서 학습 — § 10의 노트북 사용. 데이터셋 폴더 업로드 후 실행. T4 GPU 무료, 100 epochs ~30-60분. 결과: best.pt.
  4. Hailo Compiler 변환 — Hailo Developer Zone 무료 가입 → Hailo Model Zoo로 ONNX → HEF 변환: 
    bashhailomz compile yolov8n \
  --ckpt best.onnx --hw-arch hailo10h \
  --calib-set-path calibration_images/ --classes 5
  5. Pi 배포 + 자동 인식 — scp onion_mitosis_h10.hef neomscope@192.168.123.108:~/RAIM_OUTPUT/models/ → AI 패널 드롭다운에 자동 등장 → 활성화 → 라이브뷰에 한국어 분열 단계 박스.
